# ファイルサーバーハンズオン

## 目的
Windowsファイルサーバーに共有フォルダを作成し、Active Directoryのグループを利用したアクセス制御を学習する。
ユーザーへ直接権限付与するのではなく、グループを経由して権限を管理する運用方法を理解することを目的とする。

## フォルダ構成
FILE01に共有フォルダ作成。
C:\Share
  Sales
共有名：Sales

## 共有権限
Evevyone
フルコントロール
理由
実際のアクセス制御はNTFS権限で実施するため、共有権限はシンプルな設定とした。
共有権限とNTFs権限が両方設定されている場合、より厳しい権限が適用されることを学習した。

## NTFS権限
SalesフォルダのNTFS権限を設定

### 最終構成
SYSTEM          フルコントロール
Administrators  フルコントロール
GG_Sales        変更
理由
一般ユーザーには業務に必要な操作のみ許可し、権限変更や所有者変更は管理者のみ実施できるようにするため。
最小権限の原則に基づき、GG_Salesには「変更」権限を付与した。

## 継承
Salesフォルダ作成直後は親フォルダから権限が継承されていた。
そのため、Usersグループを削除しようとした際にエラーが発生。
対応手順
Sales → プロパティ → セキュリティ → 詳細設定 → 継承の無効化 → 継承されたアクセス許可を明示的なアクセス許可へ変換
その後、不要な権限を削除

## 学んだこと
Windowsのアクセス権は親フォルダから子フォルダへ継承される。
共有フォルダごとに異なる権限を設定したい場合は、軽傷を無効化して個別に管理する


## グループによる権限管理
ユーザーへ直接権限を付与するのではなく、グループへ権限を付与した。
構成
yamada → GG_Sales → Salesフォルダ
理由
移動や退職が発生した場合でも、グループの所属変更のみで権限管理ができるため

## 動作確認

### アクセス許可
ユーザー：yamada
条件：GG_Sales
結果：\\FILE01\Salesアクセス成功

### アクセス拒否
ユーザー：test02
条件：GG_Sales未所属
結果：アクセス拒否

## 学んだこと
- 共有権限とNTFS権限は両方評価され、より厳しい権限が適用される
- ユーザーではなくグループへ権限を付与する
- Windowsの権限は継承される
- 必要に応じて継承を無効化する
- 最小権限の原則に基づき権限を設定する
- ファイルサーバーの権限管理は「ユーザー → グループ → 権限」の構成で管理する
